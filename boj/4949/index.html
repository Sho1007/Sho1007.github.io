      4949. 균형잡힌 세상 / c++ / Silver4 / 12분

      문제 및 코드

      접근 방식

      Stack

      1. 괄호 문제? 스택
      2. 여는 괄호는 스택에 push
      3. 닫는 괄호가 나올 때 스택을 검사
        1. 스택이 비어있거나, 스택의 탑이 해당 괄호와 맞지 않을 때 -> false
        2. 스택이 비어있지 않고, 스택의 탑이 해당 괄호와 맞다면 -> pop
      4. 문자열 순회가 끝난 후 stack을 검사
        1. 남아있는 원소가 있다면 -> false
        2. 스택이 비어있다면 -> true

      다시 생각해 볼 점

      1. 괄호 == 스택
